Модуль 3 · Слайд 01

Шесть шагов одного клика

Нажми «Запустить» — наблюдай как клик превращается в конверсию и деньги. Каждый шаг раскрывается по очереди.

Путь от клика до выплаты
Партнёр запускает кампаниюПартнёр
Создаёт кампанию в трекере (Keitaro). Трекер генерирует уникальный tracking link с макросом для click_id.
https://track.keitaro.io/click?pid=42&offer=17&sub1={ext_id}
Пользователь кликает → рождается click_idТрекер⚡ ключевой момент
Трекер фиксирует: IP, User-Agent, гео, время, источник. Генерирует уникальный ID для этого клика. Делает редирект на лендинг, передавая click_id в URL.
→ landing.com/?clickid=a7f3k2m9&geo=NG&src=prop
Лендинг передаёт click_id рекламодателюЛендинг
Пользователь вводит номер телефона. Форма отправляет на сервер рекламодателя: номер телефона + click_id. Рекламодатель сохраняет связку: номер ↔ click_id a7f3k2m9.
POST /subscribe phone=+2348012345678 clickid=a7f3k2m9
Оператор подтверждает подпискуОператор · MTN Nigeria
Пользователь вводит PIN из SMS. MTN Nigeria проверяет: активен ли номер, есть ли деньги на балансе, нет ли блокировок. Отвечает: approved или rejected.
MTN → рекламодатель: STATUS=approved phone=+2348012345678
Рекламодатель отправляет postbackРекламодатель → Трекер⚡ S2S
HTTP GET запрос с сервера рекламодателя на URL трекера партнёра. Никакого браузера — сервер к серверу напрямую. Поэтому S2S надёжнее пикселя.
GET https://track.keitaro.io/postback?
  click_id=a7f3k2m9
  &status=approved
  &payout=0.32
  &geo=NG
Трекер засчитывает · Платформа проверяетТрекер + GG
Трекер находит клик a7f3k2m9 и помечает как конверсию. Параллельно GG Platform получает данные, антифрод проверяет паттерн. Конверсия засчитана — партнёру начислено $0.32.
✓ Конверсия засчитана click=a7f3k2m9 payout=$0.32 antifraud=CLEAN

Модуль 3 · Слайд 02

Постбек изнутри — каждый параметр

Это URL который рекламодатель отправляет на сервер трекера партнёра. Нажми на каждый параметр — узнай что он значит.

// URL постбека — рекламодатель настраивает один раз в ЛК платформы
GET https://track.keitaro.io/postback
  ?click_id={CLICK_ID}
  &status={STATUS}
  &payout={PAYOUT}
  &geo={GEO}
  &operator={OPERATOR}
click_id → a7f3k2m9
Уникальный ID клика — связывает конверсию с конкретным кликом в трекере. Без него трекер не знает кому засчитать конверсию. Самый важный параметр.
▼ нажми
status → approved / rejected
Подтверждена конверсия или нет. approved = оператор принял, деньги спишутся. rejected = оператор отклонил (нет денег, блокировка, фрод).
▼ нажми
payout → 0.32
Сумма выплаты партнёру за эту конверсию в USD. Трекер использует это для расчёта Revenue и ROI кампании. Может отличаться для разных операторов.
▼ нажми
geo → NG / KE / BR
Страна пользователя. Трекер использует для разбивки статистики по ГЕО. Помогает AM видеть какие гео дают лучший апрув и EPC.
▼ нажми

Дебаг-сценарий: найди причину

Три независимых кейса. В каждом — партнёр сообщает о проблеме. Выбери правильную гипотезу.

🔴 Кейс 1
Партнёр: «У меня в трекере 200 конверсий за вчера, в платформе — 200, но сегодня трекер показывает 80, платформа — 40. Вы занижаете!»
❌ Антифрод заблокировал половину конверсий сегодня
✅ Оператор подтверждает с задержкой — данные за незакрытый период не совпадут
❌ click_id не передаётся в постбеке
🔴 Кейс 2
Партнёр: «В трекере 150 конверсий, в платформе 75. Период закрытый — вчера. Вчера было всё нормально, сегодня утром проверяю вчерашний день и вижу такое расхождение»
❌ Задержка оператора — постбеки ещё идут
❌ Дублирование постбека в трекере
✅ Антифрод отклонил 75 конверсий — или cap оффера достигнут. Смотреть статусы в платформе.
🔴 Кейс 3
Новый партнёр, первый день. Трекер показывает 300 кликов, 0 конверсий. Период подождали 3 часа. Статус в платформе: конверсий 0. Партнёр говорит «оффер не работает»
❌ Антифрод заблокировал все конверсии
✅ Постбек не настроен — URL не добавлен в ЛК рекламодателя или настроен неверно
❌ Оффер неактивен

Трекеры: выбор инструмента

Трекер — главный инструмент AM и партнёра. От него зависит точность данных и скорость оптимизации.

⭐ Binom — выбор Golden Goose

Молниеносные инсайты, удобный интерфейс, отличная аналитика данных. Постоянная поддержка 24/7. Баланс функциональности и стоимости.

GG рекомендует Binom как основной трекер для работы с платформой.

📊 Voluum

ИИ для оптимизации кампаний, глобальные интеграции, работает бесперебойно. Высокая стоимость для большого объёма трафика.

🛡 RedTrack

Защита данных и GDPR-совместимость. Versatile для веб и мобайл. Подходит как новичкам так и профессионалам.

⚡ Bemob

Простой в использовании, гибкие тарифы. Отслеживание эффективности кампаний в реальном времени. Хорошо для старта.

🔬 PureClick

Превращает сырые данные в детальные отчёты. Мгновенные отчёты по кампаниям. Богатый набор функций.

🔧 Keitaro

Самохостинг — полный контроль над данными. Кастомизируемые лендинги без ограничений. Для опытных с IT-ресурсами.

💡 Что важно в трекере для mVAS

Скорость обработки постбеков, поддержка S2S, удобная разбивка по sub-sources, возможность создавать правила для автобана зон. Binom и Voluum закрывают все эти задачи.

3 прелендера: Слоты / Колесо фортуны / Скретч-карты. Общая концепция, разная форма.

2
Сегментация по возрасту подписки
▼
3 кампании по "subscription age" (возраст подписчика в сети):
• 0–7 дней — новые, высокий биддинг
• 8–30 дней — средние
• 30+ дней — старые, дешевле

Почему это важно: новые подписчики дороже в трафике но лучше конвертируют. Старые — дешевле, но CR ниже. Оптимальное соотношение — ключ к ROI.

3 правила оптимизации зон
▼
После первых данных по каждой зоне (источнику трафика):

🔴 Красная зона без лидов → бан
🟡 Красная с лидами (>4x стоимость лида) → снизь ставку, дай шанс
🟢 Прибыльная с лидами → повышай ставку

Не меняй несколько параметров одновременно — иначе не поймёшь что именно изменило результат.

Анализ браузеров и версий Android
▼
